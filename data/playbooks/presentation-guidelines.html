1.2 Naming Conventions

  • Use YYYYMMDD-presentation-name format for directory names
  • Use kebab-case for all naming (lowercase with hyphens)
  • Include the date in the format YYYYMMDD to allow for chronological sorting
  • Use descriptive but concise presentation names

3. Slide Design Guidelines

3.1 Common Slide Types

Each presentation should include a thoughtful mix of these slide types:

Slide Type Purpose Example Elements
Title Opening slide that introduces the topic Large title, subtitle, scripture reference
Content Presenting information, explanations Headers, paragraphs, bullet points, images
Scripture Displaying Bible passages Formatted scripture text, reference, translucent background
Discussion Prompting group interaction Question text, supporting graphics
Comparison Contrasting concepts Two-column layout, visual cues for contrast
Reflection Encouraging personal application Questions, prompts, space for contemplation

3.2 Visual Styling

Follow these visual styling principles for consistent, effective presentations:

Backgrounds

  • Use gradients for visual interest: bg-gradient-to-b from-background to-slate-100 dark:from-background dark:to-slate-950/50
  • Common gradient directions: to-b, to-r, to-br, to-bl, to-t
  • Always include dark mode variants with the dark: prefix
  • Use the base background color as a starting point for gradients

Typography

  • Main titles: text-4xl sm:text-5xl md:text-6xl font-bold
  • Section headers: text-3xl sm:text-4xl font-semibold
  • Subheadings: text-xl sm:text-2xl sm:text-3xl font-medium
  • Body text: text-base sm:text-lg md:text-xl for optimal mobile readability
  • Scripture text: font-serif text-lg sm:text-xl md:text-2xl
  • Notes or citations: text-base sm:text-lg

Layout & Spacing

  • Page padding: px-4 sm:px-6 to ensure content doesn't touch screen edges on mobile
  • Maximum content width: max-w-5xl or max-w-6xl for full slides
  • Content sections: max-w-4xl mx-auto for focused readable blocks
  • Responsive padding: p-4 sm:p-6 md:p-8 for properly scaled containers
  • Vertical spacing: space-y-4 sm:space-y-6 to optimize spacing across devices
  • Use mb-6 sm:mb-10 for responsive spacing after headings
  • Grid layouts: grid grid-cols-1 md:grid-cols-2 or grid grid-cols-1 md:grid-cols-3 to stack on mobile

Color Palette

Use these color combinations for consistency across presentations:

Purpose Default Color Dark Mode
Primary accent blue-600 blue-400
Info/Highlight boxes blue-50 blue-950/30
Success/Positive green-600 green-400
Error/Negative red-600 red-400
Warning/Caution amber-600 amber-400
Scripture background gray-100 gray-800

UI Elements

  • Cards: rounded-lg bg-[color]-50 dark:bg-[color]-950/30 shadow-md
  • Borders: border-l-4 border-[color]-500 for emphasis strips
  • Emphasis: border-b-2 border-blue-500 pb-1 for underlined headers
  • Icons: Use emojis or Lucide icons with proper sizing (h-4 w-4)

5. Best Practices

5.1 General Guidelines

  • Keep each slide focused on a single concept or idea
  • Aim for 7-12 slides for a 20-30 minute presentation
  • Include a mix of slide types (text, scripture, questions, comparisons)
  • End with reflection questions or actionable steps
  • Use comments to clearly mark each slide: {/* Title Slide */}
  • Include emoji icons to enhance visual appeal and reinforce concepts

5.2 Performance Considerations

  • Keep animations to a minimum (rely on the built-in slide transitions)
  • Use proper image optimization if including images
  • Maintain consistent max-width containers for readability
  • Use responsive design patterns (md:grid-cols-2, etc.)

5.3 Mobile Responsiveness

All presentations must be fully responsive and provide a great experience on mobile devices. Follow these mobile optimization principles:

  • Progressive enhancement: Start with mobile-friendly base styles, then enhance for larger screens using breakpoint prefixes
  • Responsive text sizing: Use smaller font sizes on mobile with scaled-up sizes for tablets and desktops 
    text-lg sm:text-xl md:text-2xl
  • Reduced padding on mobile: Use smaller padding values on mobile, larger on desktop 
    p-4 sm:p-6 md:p-8
  • Stacked layouts on mobile: Multi-column layouts should stack vertically on smaller screens 
    grid grid-cols-1 md:grid-cols-2
  • Touch-friendly targets: Ensure interactive elements are at least 44px tall/wide with adequate spacing
  • Text legibility: Use a minimum 16px (1rem) font size for body text on mobile, with adequate line spacing
  • Reduced vertical margins: Use smaller vertical margins on mobile to maximize visible content 
    mb-4 sm:mb-6 md:mb-8
  • Screen edge padding: Always include horizontal padding to prevent content from touching screen edges 
    px-4 sm:px-6
Testing: Always test your presentation on multiple device sizes before considering it complete. Ensure that all text is readable and elements maintain proper spacing on small screens.

5.4 Content Structure

Effective presentations typically follow this structure:

  1. Title/Introduction Slide: Clearly state the topic with a compelling visual
  2. Scripture Foundation: Present the key biblical text(s)
  3. Content Development: 3-5 slides explaining the topic
  4. Application: How this applies to the audience
  5. Discussion/Interaction: Questions or prompts for engagement
  6. Reflection/Conclusion: Final thoughts or actionable steps
Accessibility Tip: Use semantic HTML elements (<h1>, <h2>, etc.) and maintain good color contrast ratios. Avoid relying solely on color to convey meaning.

6. Syncing & Presentation Features

The presentation framework includes powerful syncing features:

6.1 Presentation Controls

  • Navigation: Arrow buttons for previous/next slide
  • Fullscreen: Toggle fullscreen mode
  • Sync: Presenter/viewer modes for synchronized presentations
  • Share: Generate shareable links to specific slides

6.2 Presentation Modes

  • Presenter Mode: Control the slides that others see
  • Follower Mode: View slides controlled by the presenter
  • Solo Mode: Navigate independently even during a presentation
Important: Always set a unique, predictable presentationKey to ensure proper syncing. This should be derived from your presentation name and be consistent across sessions.

7. Technical Reference

7.1 Core Components

  • PresentationContainer: Wrapper component with syncing capabilities
  • PresentationControls: Navigation and control UI
  • Slide: Individual slide component with specified index

7.2 Required Props

  • totalSlides: The total number of slides in the presentation
  • presentationKey: A unique identifier for the presentation
  • index: The slide number (starting from 1)

7.3 Optional Props

  • className: Additional CSS classes for customization

9. Publishing Your Presentation

9.1 Registering in Recent Sharings

After creating your presentation, you must register it in the "Recent Sharings" component to make it visible on the presentations index page.

  1. Open apps/webapp/src/components/presentations-grid.tsx
  2. Add your presentation to the PRESENTATIONS array at the top of the file
  3. Follow this template: 
    {
  id: 'your-presentation-id',
  title: 'Your Presentation Title',
  date: 'Month DD, YYYY',
  description: 'Brief description of your presentation',
  path: '/presentations/YYYYMMDD-presentation-name',
}
  4. Place newer presentations at the top of the array to maintain chronological order
  5. Make sure the path matches your presentation's directory name
Example: For a presentation called "Making a Difference" created on May 16, 2025, located at /presentations/20250516-luke-10: 
{
  id: 'luke-10',
  title: 'Making a Difference',
  date: 'May 16, 2025',
  description: 'Cell Group sharing on Luke 10: The Good Samaritan and Martha & Mary',
  path: '/presentations/20250516-luke-10',
}

After adding your presentation to the array, it will automatically appear in the presentations index page, making it discoverable for users.
